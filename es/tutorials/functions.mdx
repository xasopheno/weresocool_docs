Si has estado haciendo esto en orden, no tenías idea hasta que
ahora, ¡esas funciones se han estado escondiendo a simple vista!
cualquier bloque excepto el bloque "main" se puede extender a
tomar argumentos.

<WereSoCool language={
`{ f: 261.6, l: 1/3, g: 1, p: 0 }\n
melody(a) = {
  a | Repeat 3
}\n
main = {
  melody(
    Seq [
      Fm 5/4, Fm 9/8, Fm 1
    ]
  )
}`
}/>

Las funciones son muy potentes. Algunas de las piezas de Bach más sencillas pueden describirse con bastante claridad como un pequeño conjunto de funciones que se ejecutan
una serie de entradas diferentes. Aquí hay un ejemplo más complicado.


<WereSoCool language={
`{ f: 220, l: 7/8, g: 1/5, p: 0 }\n
overtones = {
  Overlay [
    {7, 0, 1/3, 1},
    {5, 0, 1/3, -1},
    {3, 0, 1, 1},
    {2, 0, 1, -1},
    {1/1, 2, 1, 1},
    {1/1, 0, 1, -1},
    {1/2, 0, 1, 0},
    {1/2, 1, 1, 0},
  ]
}\n
bass_overtones = {
  Overlay [
    {4, 0, 1, 0},
    {4, 0, 1, 0},
    {1/1, 2, 1, 1},
    {1/1, 0, 1, -1},
    {1/2, 2, 1/2, 1},
    {1/2, 0, 1/2, -1},
  ]
}\n
-- Esta es una función que toma 4 argumentos y
-- luego los reproduce en un nuevo orden.
cool_function(a, b, c, d) = {
  Overlay [
    overtones | 
    Seq [
      c, b, a, b, c, d, c | Lm 2
    ],
    -- y también hace una segunda melodía genial
    bass_overtones
    | Seq [
      Fm 0,
      a | Fm 1/2,
      a | Fm 3/4, 
      c | Fm 3/4, 
    ] | Lm 2
  ]
}\n
main = {
  Seq [
    -- Y luego podemos usar esa función con diferentes
    -- Argumentos para generar diferentes melodías.
    cool_function(Fm 1, Fm 9/8, Fm 5/4, Fm 3/2) | Repeat 2, 
    cool_function(Fm 1, Fm 5/4, Fm 3/2, Fm 15/8) | Repeat 2,
    cool_function(Fm 7/6, Fm 9/8, Fm 1, Fm 7/8) | Repeat 2,
    cool_function(Fm 3/2, Fm 3/4, Fm 5/4, Fm 1) | Repeat 2
  ]
  -- Usando Seq, podemos armar rápidamente
  -- una estructura razonable para una canción.
  | Seq [Fm 1, Fm 4/3, Fm 7/6, Fm 1]
  | Lm 1/7 
}`
}/>
