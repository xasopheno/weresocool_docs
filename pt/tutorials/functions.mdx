Se você estava fazendo esses tutoriais em ordem, você não tinha ideia até
agora, essas funções estão escondidas à vista de todos!
Qualquer bloco, exceto o bloco "main", pode ser estendido para
aceitar argumentos.

<WereSoCool language={
`{ f: 261.6, l: 1/3, g: 1, p: 0 }\n
melody(a) = {
  a | Repeat 3
}\n
main = {
  melody(
    Seq [
      Fm 5/4, Fm 9/8, Fm 1
    ]
  )
}`
}/>

As funções são muito poderosas. Algumas das peças mais simples de Bach podem ser descritas claramente como um pequeno conjunto de funções executadas
uma série de entradas diferentes. Aqui está um exemplo mais complicado.

<WereSoCool language={
`{ f: 220, l: 7/8, g: 1/5, p: 0 }\n
overtones = {
  Overlay [
    {7, 0, 1/3, 1},
    {5, 0, 1/3, -1},
    {3, 0, 1, 1},
    {2, 0, 1, -1},
    {1/1, 2, 1, 1},
    {1/1, 0, 1, -1},
    {1/2, 0, 1, 0},
    {1/2, 1, 1, 0},
  ]
}\n
bass_overtones = {
  Overlay [
    {4, 0, 1, 0},
    {4, 0, 1, 0},
    {1/1, 2, 1, 1},
    {1/1, 0, 1, -1},
    {1/2, 2, 1/2, 1},
    {1/2, 0, 1/2, -1},
  ]
}\n
-- Esta é uma função que recebe 4 argumentos e
-- então os reproduz em uma nova ordem.
cool_function(a, b, c, d) = {
  Overlay [
    overtones | 
    Seq [
      c, b, a, b, c, d, c | Lm 2
    ],
    -- e também faz uma segunda melodia legal
    bass_overtones
    | Seq [
      Fm 0,
      a | Fm 1/2,
      a | Fm 3/4, 
      c | Fm 3/4, 
    ] | Lm 2
  ]
}\n
main = {
  Seq [
    -- E então podemos usar essa função com diferentes
    -- argumentos para gerar diferentes melodias.
    cool_function(Fm 1, Fm 9/8, Fm 5/4, Fm 3/2) | Repeat 2, 
    cool_function(Fm 1, Fm 5/4, Fm 3/2, Fm 15/8) | Repeat 2,
    cool_function(Fm 7/6, Fm 9/8, Fm 1, Fm 7/8) | Repeat 2,
    cool_function(Fm 3/2, Fm 3/4, Fm 5/4, Fm 1) | Repeat 2
  ]
  -- E podemos usar uma sequência para 
  -- criar uma forma de música
  | Seq [Fm 1, Fm 4/3, Fm 7/6, Fm 1]
  | Lm 1/7 
}`
}/>
